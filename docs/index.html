Documentación de Git y Git Flow

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear cambios en el código fuente durante el desarrollo de software. Fue creado por Linus Torvalds en 2005.

¿Qué es Git Flow?

Git Flow es un flujo de trabajo de ramificación para Git, propuesto por Vincent Driessen. Proporciona una estructura robusta para gestionar las ramas en un proyecto de desarrollo de software, facilitando la colaboración y el mantenimiento del código.

¿Por qué utilizamos Git y Git Flow en el desarrollo?

Porque Git y Git Flow mejora la colaboración entre los desarrolladores, mantiene un historial claro de los cambios en el código y facilita la gestión de versiones y lanzamientos de software.

Descripción paso a paso del proyecto

Paso 1

Inicializamos el repositorio con git init, creamos el README.md y .gitignore del proyecto con touch, hacemos un git add y el primer commit para despues conectarnos al repositorio creado previamente en GitHub y finalizar el push en la rama main.

Paso 2

Inicializamos el repositorio Git para usar el flujo de trabajo Git Flow.

Paso 3

Iniciamos la feature estructura_inicial y creamos el index.html con touch.

Paso 4

Git add y commit desde el usuari 1 para luego cerrar la feature estructura_inicial.

Paso 5

Hacemos un push a develop para actualizar los cambios.

Paso 6

Iniciamos la feature modificar_atributos, el usuario 2 modifica el html, lo añade y comitea.

Paso 7

Finalizamos la feature modificar_atributos y hacemos un push a la rama develop.

Paso 8

Iniciamos la feature modificar_contenido, el usuario 2 modifica el htlm, git add y commit antes de cerrarla.

Paso 9

Actualizamos los cambios con un push a la rama develop.

Paso 10

Iniciamos la feature estilos-css, modificamos los estilos del html con el usuario 3, hacemos un git add y antes de cerrar la feature un commit, al terminar subimos los cambios a develop con un push.

Paso 11

Iniciamos la release v1.0.

Paso 12

Terminamos la release v1.0.

Paso 13

Actualizamos con push la rama main y develop.

Paso 14

Iniciamos el hotfix, una vez arreglado el error continuamos con git add, comiteamos y cerramos el hotfix.

Paso 15

Hacemos un git push a main.

Último paso

Terminamos con un git push a la rama develop.
